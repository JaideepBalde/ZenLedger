Project Details: ZenLedger - by JD

Overview:
ZenLedger is a comprehensive family financial governance platform built for secure, institutional-grade management of household finances. It operates on a "cluster" model where a HOST (parent) manages MEMBER (family member) accounts, enabling real-time transaction tracking, budget allocation, secure messaging, and capital requests. Integrated AI provides fiscal audits and wealth strategies, while a unique focus engine plays mantra audio for productivity. The app emphasizes security, user experience, and family collaboration.

Architecture:
- Frontend: Single-page React application using TypeScript, Vite for build tooling, and Tailwind CSS for styling. Components include Onboarding, HelpSection, and main App with tabs for Ledger, Registry, Messages, and Requests.
- Backend: Express.js server running on Node.js, handling API endpoints for authentication, users, transactions, requests, messages, and audit logs. Uses MongoDB with Mongoose for data persistence.
- Storage: Local storage for session management; API calls to backend for data synchronization.
- Deployment: Configured for Vercel, with environment variables for API keys.

Tech Stack:
- Frontend: React 19.2.3, TypeScript ~5.8.2, Vite 6.2.0, Tailwind CSS (via custom styles)
- Backend: Express 5.2.1, Node.js (via @types/node 22.14.0), MongoDB with Mongoose 9.1.2
- Authentication: JSON Web Tokens (jsonwebtoken 9.0.3) for session management
- AI: Google GenAI (@google/genai 1.34.0) for Gemini API integration
- Other: CORS 2.8.5 for cross-origin requests, audio playback for mantra.mp3

Key Features:
- Ledger: Real-time transaction logging with categories (Allocation, Operations, Recreation, Development, Social, Reserve, Grant, Miscellaneous), types (CREDIT/DEBIT), and descriptions.
- Registry: HOST provisions MEMBER accounts, allocates funds, and manages liquidity.
- Messages: Encrypted broadcasts to cluster or direct node-to-node communications.
- Requests: MEMBERS submit capital requests with reasons; HOST approves/rejects and executes transactions.
- AI Fiscal Audit: Analyzes transaction data to suggest wealth strategies using Gemini API.
- Focus Engine: Plays local mantra.mp3 audio with interactive controls for productivity.
- Onboarding: Guided setup for new users.
- Audit Logs: Tracks actions like node creation and ledger updates.

Database Schema:
- Users: id, familyId, username, password (hashed), role (HOST/MEMBER), isActive
- Transactions: id, userId, amount, type, category, description, timestamp
- MoneyRequests: id, childId, amount, reason, status (PENDING/APPROVED/REJECTED), timestamp
- FamilyMessages: id, familyId, fromId, toId ('cluster' or userId), text, timestamp, isRead
- AuditEntries: action, metadata, timestamp

Security:
- JWT-based authentication with 24-hour expiration.
- Password hashing (assumed in UserModel).
- Role-based access control (HOST vs MEMBER).
- Secure API headers and error handling.

Setup Instructions:
1. Clone repository and navigate to project directory.
2. Install dependencies: npm install
3. Create .env.local with GEMINI_API_KEY.
4. Start backend: npm run start (runs tsx server.ts on port 3001)
5. Start frontend: npm run dev (Vite dev server)
6. Build for production: npm run build; npm run preview

Deployment:
- Vercel configuration in vercel.json.
- Accessible via AI Studio: https://ai.studio/apps/drive/18HXUJSuB--5oBsI7GkSxCE07wVs50GHa
